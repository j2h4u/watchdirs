# Codebase Structure

**Analysis Date:** 2026-06-17

## Directory Layout

```text
watchdirs/
├── AGENTS.md              # repo-specific operating constraints
├── README.md              # product and usage overview
├── examples/              # sample config files
├── ops/systemd/           # systemd services, timers, and socket units
├── src/watchdirs/         # runtime package
├── tests/                 # unit, integration, and benchmark tests
└── pyproject.toml         # build, lint, type, import, and test config
```

## Directory Purposes

**`src/watchdirs/`:**
- Purpose: production package code.
- Contains: CLI, collection, database, diagnostics, reporting, config, and shared models.
- Key files: `src/watchdirs/cli.py`, `src/watchdirs/models.py`, `src/watchdirs/db/schema.sql`.

**`src/watchdirs/collect/`:**
- Purpose: filesystem walking and mount policy.
- Contains: scanner, mountinfo parsing, mount classification.
- Key files: `src/watchdirs/collect/scanner.py`, `src/watchdirs/collect/mounts.py`, `src/watchdirs/collect/classify.py`.

**`src/watchdirs/db/`:**
- Purpose: SQLite lifecycle and retention.
- Contains: connection helpers, migrations, retention, schema SQL.
- Key files: `src/watchdirs/db/connection.py`, `src/watchdirs/db/migrations.py`, `src/watchdirs/db/retention.py`.

**`src/watchdirs/reporting/`:**
- Purpose: query and render growth/report outputs.
- Contains: query builders, diff/frontier logic, text/JSON renderers.
- Key files: `src/watchdirs/reporting/queries.py`, `src/watchdirs/reporting/render.py`, `src/watchdirs/reporting/frontier.py`.

**`src/watchdirs/diagnostics/`:**
- Purpose: cross-check index totals against host signals.
- Contains: df-vs-index, deleted-open, Docker/containerd diagnostics, compact summary.
- Key files: `src/watchdirs/diagnostics/df_index.py`, `src/watchdirs/diagnostics/deleted_open.py`, `src/watchdirs/diagnostics/docker.py`, `src/watchdirs/diagnostics/summary.py`.

**`ops/systemd/`:**
- Purpose: packaged systemd units for unattended operation.
- Contains: collect/prune/vacuum services and timers plus the query socket.
- Key files: `ops/systemd/watchdirs-collect.service`, `ops/systemd/watchdirs-collect.timer`, `ops/systemd/watchdirs-prune.service`, `ops/systemd/watchdirs-vacuum.service`, `ops/systemd/watchdirs-query.socket`.

**`examples/`:**
- Purpose: runnable configuration examples.
- Contains: host-facing TOML example.
- Key files: `examples/host.watchdirs.toml`.

**`tests/`:**
- Purpose: behavior lockfiles for runtime, CLI, DB, and systemd assets.
- Contains: `pytest` suites plus benchmark-style checks.
- Key files: `tests/test_cli_collect.py`, `tests/test_mount_policy.py`, `tests/test_systemd_units.py`, `tests/test_reporting_queries.py`.

## Key File Locations

**Entry Points:**
- `src/watchdirs/__main__.py`: `python -m watchdirs`.
- `src/watchdirs/cli.py`: command dispatcher and socket entry.

**Configuration:**
- `pyproject.toml`: package metadata, linting, type checking, import contracts, coverage, pytest config.
- `examples/host.watchdirs.toml`: sample host config.

**Core Logic:**
- `src/watchdirs/collect/scanner.py`: recursive directory snapshotting.
- `src/watchdirs/db/migrations.py`: snapshot insert/update and schema initialization.
- `src/watchdirs/reporting/queries.py`: report row selection and pairing.
- `src/watchdirs/diagnostics/df_index.py`: df-vs-index analysis.

**Testing:**
- `tests/`: all test files.
- `tests/bench/`: benchmark-oriented checks for churn and size behavior.

## Naming Conventions

**Files:**
- Module names are lowercase with underscores, grouped by domain: `collect`, `diagnostics`, `reporting`, `db`.
- Test files use `test_*.py` and mirror the behavior being checked: `tests/test_ops_retention.py`.

**Directories:**
- Feature areas are plural or domain-based: `collect`, `diagnostics`, `reporting`, `tests`.
- Operational assets live outside `src/` in `ops/systemd/` and `examples/`.

## Where to Add New Code

**New Feature:**
- Primary code: `src/watchdirs/<domain>/` if it belongs to an existing layer, or a new submodule under `src/watchdirs/` if the layer is distinct.
- Tests: `tests/test_<feature>.py` or the closest existing suite in `tests/`.

**New Component/Module:**
- Implementation: place in the matching layer directory, for example `src/watchdirs/diagnostics/` for host cross-checks or `src/watchdirs/reporting/` for read-only query logic.

**Utilities:**
- Shared helpers: `src/watchdirs/models.py` for dataclasses and row-shapes, `src/watchdirs/config.py` for config-related helpers.

**Systemd integration:**
- Units/timers/sockets: `ops/systemd/`.
- Keep unit semantics in repo-owned files rather than ad hoc scripts.

## Special Directories

**`src/watchdirs.egg-info/`:**
- Purpose: packaging metadata generated by the build/install flow.
- Generated: Yes.
- Committed: Present in tree; treat as generated package metadata.

**`.planning/`:**
- Purpose: repo-local planning and documentation artifacts.
- Generated: Partly.
- Committed: Yes, but keep analysis docs scoped to the GSD workflow.

**`.venv/`, `.pytest_cache/`, `.ruff_cache/`, `.import_linter_cache/`:**
- Purpose: local tooling caches and virtual environment.
- Generated: Yes.
- Committed: No.
