<threat_model>
## Trust Boundaries

| Boundary | Description |
| --- | --- |
| SQLite rows -> reporting dataclasses | Stored collapse metadata becomes agent-facing output. |
| Requested path -> explain-path lookup | User-supplied path may target a descendant that is intentionally not indexed. |
| Raw path bytes -> JSON/text renderers | Filesystem bytes are decoded only at display boundaries. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-03.2-09 | Information Disclosure | render null metadata | mitigate | Render collapse fields conditionally only for collapsed rows; tests assert ordinary rows omit those keys. |
| T-03.2-10 | Tampering | query top-child joins | mitigate | Join `top_child_id` through `paths` and materialize bytes; no interpolated path SQL. |
| T-03.2-11 | Denial of Service | explain-path inside collapsed subtree | mitigate | Return deepest indexed collapsed ancestor instead of requiring absent descendant rows. |
| T-03.2-12 | Spoofing | collapsed-row agent interpretation | mitigate | Payload includes `collapsed: true`, reason, folded directory count, and top-child bytes so opaque rows are explicit. |
| T-03.2-SC | Tampering | package installs | accept | No package-manager install tasks exist in this plan. |
</threat_model>
