# Phase 03.2: Scan-Time Folder Collapse Verification Report

**Phase Goal:** Cut the directory cardinality that drives DB size by collapsing never-interesting subtrees into a single aggregate row at scan time, preserving the collapsed directory's recursive size while dropping descendant rows. The phase also needed explicit collapse metadata, reporting visibility, no-collapse regression coverage, and a product-schema storage proof.
**Verified:** 2026-06-17
**Status:** passed
**Re-verification:** No - initial verification after code-review fixes

The phase goal is achieved. Collapse is now a scan-time row-emission policy with schema-v4 metadata, reporting support, explain-path fallback for folded descendants, and benchmark proof that collapsed scan output stores fewer rows and fewer SQLite bytes than expanded output.
