# Phase 03.2 Cross-AI Plan Reviews

## Cycle 1 Summary

Two unresolved HIGH findings remain. The plans are strong overall, but reviewers found correctness interactions at the boundaries between collapse and existing filesystem/reporting invariants that must be made executable in PLAN.md before `/gsd-execute-phase`.

### Current HIGH Concerns

- **H1 — Storage-domain totals may be wrong with collapsed rows.** OpenCode flagged that `query_indexed_storage_domain_totals()` is not mentioned in the plans. Collapsed rows keep recursive bytes while descendant rows are absent, and the existing parent/domain walk may misattribute a collapsed subtree when intermediate rows are missing. PLAN.md needs a task or acceptance criterion that audits/tests collapsed-row interaction with storage-domain totals.
- **H2 — `collapse.never` can be defeated by a collapsing ancestor.** Claude flagged that plans only block collapse when the current node itself matches `collapse.never`. If a protected path is nested under an ancestor that qualifies by name/fan-out/descendant-count, row-buffer truncation can still drop the protected subtree. PLAN.md needs an explicit semantic decision and tests. Recommended decision: a `never` path poisons ancestors on its path, so any ancestor containing a protected descendant stays expanded.

### Current Actionable Non-HIGH Concerns

- **A1 — Preserve or explicitly surface partial-error/skipped-mount evidence under collapsed subtrees.** Claude flagged that `del rows[frame.row_start:]` can delete skipped-mount or partial-error evidence rows inside a collapsed subtree, undermining FSEM-05. PLAN.md needs a tested decision: preserve evidence rows, summarize evidence on the boundary row, or explicitly document/test that collapse hides descendant evidence.
- **A2 — Prove migration idempotency and interruption behavior.** Claude flagged that the v3-to-v4 migration plan does not test rerun/partial-state behavior. PLAN.md should require tests that re-running migration succeeds and that partial/duplicate-column states cannot advance `user_version` incorrectly.
- **A3 — Add an end-to-end scanner-to-storage size proof.** Claude flagged that Plan 04's storage proof constructs collapsed rows by hand. PLAN.md should require at least one synthetic filesystem fixture scanned with real `scan_root()` under collapse and no-collapse policies, then inserted through product SQLite paths to prove row-count reduction and aggregate equality.
- **A4 — Stress-test row-buffer contiguity with mixed subtrees.** Claude flagged that the contiguous-suffix assumption behind `del rows[frame.row_start:]` is load-bearing. PLAN.md should require a mixed subtree test with normal child dirs plus skipped/error rows, tied to the intended evidence behavior from A1.
- **A5 — Pin path-component-aware `collapse.never` matching.** Claude flagged substring risk such as `/data` matching `/database`. PLAN.md should require exact-or-descendant path-component semantics and a `/data` vs `/database` test.
- **A6 — Clarify collapsed `top_child` chain semantics.** OpenCode noted that a collapsed row's `top_child` may itself be collapsed. PLAN.md should document this as intentional, if accepted, and ensure explain/report payloads remain understandable.
- **A7 — Avoid inefficient or incomplete explain-path collapsed-ancestor lookup.** OpenCode noted that explain-path boundary resolution may scan a limited result set. PLAN.md should specify a direct query for the deepest collapsed ancestor or otherwise prove the existing query path is complete enough.
- **A8 — Handle changed collapse policy across snapshots in diff metadata.** OpenCode flagged current-vs-baseline metadata fallback edge cases when a path is collapsed in one snapshot and expanded in another. PLAN.md should add an acceptance criterion or test for current-first/baseline fallback semantics.
- **A9 — Keep collapse size proof separate from the old blob-vs-dictionary benchmark.** OpenCode noted the benchmark helper must compare collapsed and uncollapsed product-schema DBs only. PLAN.md mostly says this, but the validation should assert it.
- **A10 — Add exact `collapsed_dirs` threshold/count convention tests.** Claude flagged possible off-by-one ambiguity around descendant count and stored `collapsed_dirs`. PLAN.md should pin whether `frame.dir_count` excludes the collapsed directory and test threshold equality.
- **A11 — Trim per-plan artifact lists to each plan's real write scope.** Claude flagged that every plan currently repeats the whole phase artifact list, which can mislead executors and source grounding. PLAN.md files should list only artifacts produced by that specific plan, while a phase-wide audit can remain elsewhere.

## OpenCode Review

**Overall risk:** LOW.

OpenCode found the plan sequence strong and migration discipline sound. It raised one HIGH concern:

- `query_indexed_storage_domain_totals()` may misattribute collapsed subtree bytes when intermediate rows are absent; add a domain-total collapse audit test or a Plan 03 task.

Actionable non-HIGH OpenCode findings:

- Document that `top_child` can point to another collapsed row.
- Ensure explain-path collapsed-boundary lookup is complete, not limited by a broad result set.
- Test diff metadata when collapse policies differ between snapshots.
- Keep product-schema collapse size proof separate from old blob-vs-dictionary benchmark comparisons.

## Claude Review

**Overall risk:** MEDIUM.

Claude found one HIGH correctness gap:

- `collapse.never` must protect nested paths even when an ancestor independently qualifies for collapse, or the limitation must be explicitly documented. Recommended: a protected descendant prevents ancestor collapse.

Actionable non-HIGH Claude findings:

- Decide and test how collapse interacts with partial-error and skipped-mount evidence rows.
- Test migration idempotency and interruption/partial-state behavior.
- Add an end-to-end `scan_root()` to SQLite storage-size proof.
- Stress-test row-buffer truncation with mixed child/error/skipped rows.
- Pin path-component-aware `never` matching.
- Trim per-plan artifact lists to each plan's true write scope.
- Document direct-child-only `top_child` limitation when a direct file dominates.
- State v1/v2 DB behavior explicitly if only v3 and v0 bootstrap are supported.
- Pin `collapsed_dirs` off-by-one semantics.
