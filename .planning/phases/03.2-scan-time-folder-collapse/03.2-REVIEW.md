## Summary

Reviewed the Phase 03.2 scanner, schema, reporting, benchmark, and regression-test changes. The phase ships two core correctness regressions: `explain-path` crashes for targets inside folded subtrees, and config validation still permits roots that traverse a symlinked ancestor even though the scanner is meant to avoid following symlinks. The new benchmark helpers also produce structurally invalid synthetic snapshots under churn and drop persisted collapse metadata when loading real scans.

## Narrative Findings (AI reviewer)

## Critical Issues

### CR-01: `explain-path` crashes on paths inside collapsed subtrees

**Classification:** BLOCKER
**File:** `src/watchdirs/reporting/queries.py:562-571`
**Issue:** The new collapsed-ancestor fallback returns only the collapsed ancestor row when the requested target path is hidden by scan-time collapse. `run_explain_path()` then passes the original target path into `explain_path_breakdown()`, which still does `next(row for row in rows if row.path == target_path)` in `src/watchdirs/reporting/frontier.py:113`. That raises `StopIteration` for every request like `watchdirs explain-path /srv/cache/deep/file.txt` when `/srv/cache` is collapsed, so the advertised fallback is not actually usable from the CLI.
**Fix:**
```python
# Option 1: make the fallback return both the matched row set and the effective target.
rows, effective_target_path = query_explain_path_rows(...)
breakdown = explain_path_breakdown(
    rows,
    target_path=effective_target_path,
    limit=effective_limit,
    depth=effective_depth,
)
```

### CR-02: Config validation still allows roots that traverse a symlinked ancestor

**Classification:** BLOCKER
**File:** `src/watchdirs/config.py:103-109`
**Issue:** `validate_roots()` rejects only a root path whose final component is a symlink. A configured root like `/tmp/link/subdir` is accepted when `link` is a symlink to another tree, and `scan_root()` will then read the real target through that symlinked ancestor. That violates the project’s filesystem-safety contract to avoid following symlinks and can make the scanner walk an unintended tree outside the operator’s declared root.
**Fix:**
```python
def _reject_symlink_ancestors(path: Path) -> None:
    current = Path(path.anchor)
    for part in path.parts[1:]:
        current /= part
        if current.is_symlink():
            raise ConfigError(
                "symlink_root",
                str(path),
                "configured root must not traverse a symlinked path component",
            )
```

## Warnings

### WR-01: Synthetic churn rewrites paths without repairing parent/top-child relationships

**Classification:** WARNING
**File:** `src/watchdirs/bench/size.py:205-235`
**Issue:** `replicate_snapshots()` rotates the first `rotate` rows onto fresh paths, but `_with_path()` updates only `row.path`. `parent_path` and `top_child_path` are left pointing at the old tree, and the root row itself can be rotated away from the snapshot root. With `churn > 0`, the benchmark stops modeling a valid directory hierarchy and starts measuring broken synthetic rows, so the row-count and SQLite-size comparisons are no longer representative.
**Fix:**
```python
# Rotate complete subtrees, not individual rows, and rewrite every affected path field.
# At minimum:
# - never rotate the root row
# - rewrite parent_path for rotated descendants
# - rewrite top_child_path when it points inside the rotated subtree
```

### WR-02: Real-scan benchmark loader drops the persisted collapse metadata

**Classification:** WARNING
**File:** `src/watchdirs/bench/size.py:480-520`
**Issue:** `_load_real_scan_rows()` reads only the pre-03.2 columns and reconstructs every `DirectoryAggregate` with `collapsed=False` plus empty `collapse_reason`, `collapsed_dirs`, and `top_child_*`. If the source database already contains collapsed rows, the benchmark silently rebuilds a different schema-v4 payload than the product stored, understating the real on-disk cost of collapsed metadata and invalidating “measure a real scan DB” runs.
**Fix:**
```python
SELECT
    p.path AS path,
    pp.path AS parent_path,
    d.depth,
    d.apparent_bytes,
    d.disk_bytes,
    d.file_count,
    d.dir_count,
    d.error,
    d.collapsed,
    d.collapse_reason,
    d.collapsed_dirs,
    tp.path AS top_child_path,
    d.top_child_disk_bytes
FROM directory_sizes d
JOIN paths p ON p.id = d.path_id
LEFT JOIN paths pp ON pp.id = d.parent_id
LEFT JOIN paths tp ON tp.id = d.top_child_id
```
