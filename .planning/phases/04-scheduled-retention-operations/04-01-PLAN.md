<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| CLI args -> filesystem lock path | `--db` controls the database path, and the lock path is derived from it. |
| Concurrent process -> SQLite writer | Another process may already be collecting, pruning, or vacuuming the same database. |
| CLI error -> stdout/stderr/journal | Lock errors become JSON or stderr records visible to agents and operators. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-04-01-01 | Tampering | `run_collect` SQLite writes | mitigate | Acquire the FD-held exclusive lock before opening/writing the database; fail before snapshot creation on contention. |
| T-04-01-02 | Denial of Service | lock acquisition | mitigate | Use non-blocking `LOCK_NB`; do not sleep, retry indefinitely, or wait behind another writer. |
| T-04-01-03 | Repudiation | lock-conflict evidence | mitigate | Return stable `operation_locked` JSON and stderr context including database and lock paths. |
| T-04-01-04 | Information Disclosure | local path in error context | accept | Local forensic CLI already exposes configured paths; lock errors do not include file contents or snapshot data. |
</threat_model>
