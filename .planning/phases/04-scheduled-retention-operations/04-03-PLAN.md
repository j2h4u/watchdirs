<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| CLI args -> maintenance DB path | Operator-selected DB path is opened for heavy SQLite maintenance. |
| Maintenance command -> SQLite file | `VACUUM` rewrites the SQLite database and interacts with WAL sidecars. |
| Concurrent process -> maintenance lock | Another writer may be collecting or pruning while maintenance is requested. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-04-03-01 | Denial of Service | `VACUUM` runtime and disk pressure | mitigate | Keep vacuum as an explicit command and later weekly timer; before running it, report advisory free-space risk from SQLite page counts and `os.statvfs`. |
| T-04-03-02 | Tampering | concurrent SQLite writer during vacuum | mitigate | Acquire the shared operation lock before opening/running maintenance. |
| T-04-03-03 | Repudiation | maintenance result | mitigate | Emit before/after counters and database path in JSON/text output. |
| T-04-03-04 | Denial of Service | WAL/busy behavior | mitigate | Reuse `open_connection` busy timeout, run `wal_checkpoint(TRUNCATE)` after successful vacuum, and expose busy/log/checkpointed result fields plus a warning on busy or partial checkpoint. |
</threat_model>
