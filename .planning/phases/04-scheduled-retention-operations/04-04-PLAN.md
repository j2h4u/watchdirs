## Trust Boundaries

| Boundary | Description |
|---|---|
| systemd timer -> CLI process | Timers launch privileged local CLI work with fixed arguments. |
| unit file -> host filesystem | Services read configured roots and write `/var/lib/watchdirs/watchdirs.sqlite3`. |
| README commands -> operator action | Docs guide enabling, checking, and diagnosing the live timers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-04-04-01 | Elevation of Privilege | systemd service ExecStart | mitigate | Use fixed absolute `/usr/local/bin/watchdirs` argv with no shell interpolation and document expected command, config, and database paths. |
| T-04-04-02 | Denial of Service | scheduled collect workload | mitigate | Pin `Nice=19`, `IOSchedulingClass=best-effort`, and `IOSchedulingPriority=7` in service files. |
| T-04-04-03 | Repudiation | missed or failed timer runs | mitigate | Set `Persistent=true` timers and document `systemctl status` plus `journalctl` verification commands. |
| T-04-04-04 | Tampering | accidental cleanup coupling | mitigate | Keep unit files and docs scoped to collect/prune/vacuum only per D-12. |
| T-04-04-05 | Information Disclosure | journal output | accept | The tool is local and already reports host paths; docs should not include secrets or file contents. |
</threat_model>
