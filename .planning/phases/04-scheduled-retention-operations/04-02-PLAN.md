<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| CLI args -> retention policy | Operator-provided `--hourly-days` and `--daily-days` control delete eligibility. |
| Retention selector -> SQLite delete | Application code chooses snapshot IDs; SQLite executes destructive deletes. |
| Snapshot delete -> path dictionary GC | Cascades remove child rows, then app code removes unreferenced dictionary rows. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-04-02-01 | Tampering | `prune_snapshots` delete set | mitigate | Validate policy windows as positive integers, compute keep-set before delete, and delete only snapshot IDs not in the keep-set. |
| T-04-02-02 | Tampering | child-table retention | mitigate | Rely on `snapshots` FK cascades; do not issue direct deletes against `directory_sizes` or `snapshot_mounts`. |
| T-04-02-03 | Repudiation | prune result | mitigate | Return JSON counts and deleted snapshot IDs so operators can see exactly what changed. |
| T-04-02-04 | Denial of Service | concurrent prune/collect | mitigate | Acquire the shared non-blocking operation lock before opening the mutating retention transaction. |
</threat_model>
