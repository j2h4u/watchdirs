# Phase 03.1 Plan 03: Reporting Wiring onto the Path Dictionary Summary

Wired every path-reading report SELECT onto the Plan-02 flat dictionary: one `JOIN paths` per query, the cross-snapshot diff converted to integer `path_id` equality with an `all_ids` CTE, and the docker/containerd hint GLOB kept working under the dictionary while still binding raw byte prefixes. A golden-equivalence harness pins diff/top/storage-domain output (including a non-UTF-8 path round-trip) so the SQL-shape change is provably behavior-preserving. The suite went from 109 passed / 70 failed to **184 passed / 0 failed**.

## What Was Built

- **`reporting/queries.py`** — three SELECT rewrites, output columns and ORDER BY clauses preserved verbatim:
  - `query_indexed_storage_domain_totals`: `JOIN paths p ON p.id = ds.path_id` + `LEFT JOIN paths pp ON pp.id = ds.parent_id`, selecting `p.path AS path, pp.path AS parent_path`.
  - `query_top_rows`: `JOIN paths p ON p.id = ds.path_id`, selects `p.path`, orders `ds.disk_bytes DESC, p.path ASC`.
  - `query_diff_rows`: `all_ids` CTE over `path_id` (UNION of both snapshots), `prev.path_id = a.path_id` / `curr.path_id = a.path_id` int-equality joins, `JOIN paths p ON p.id = a.path_id`, parent joins `LEFT JOIN paths pp/cp` with `COALESCE(cp.path, pp.path) AS parent_path`. Classification CASE switched from `prev.path/curr.path IS NULL` to `prev.path_id/curr.path_id IS NULL`. The row-materialization loop and all `bytes(row[...])` consumers are unchanged.
- **`cli.py`** — `_collect_indexed_docker_path_hints` query rewritten to `SELECT p.path FROM directory_sizes ds JOIN paths p ON p.id = ds.path_id WHERE ds.snapshot_id = ? AND (p.path = ? OR p.path GLOB ?)`, keeping the existing `(snapshot.id, prefix, prefix + b"/*")` binding so the GLOB still binds raw bytes.
- **`tests/test_reporting_equivalence.py`** (new) — 4 tests: golden diff (all five classifications + exact ORDER BY), non-UTF-8 diff round-trip, golden top (order + disk bytes), and a two-domain storage layout with nested-submount subtraction (`ext4` root-fs 100−40=60, `xfs` submount 40). Golden values are hand-derived constants, reusing the canonical `_open_db`/`_seed_snapshot`/`_directory_row`/`_mount` fixtures from `test_reporting_queries`.
- **`tests/test_diagnostics_docker.py`** — DB-backed regression `test_collect_indexed_docker_path_hints_resolves_via_dictionary_join`: builds a fixture DB with `/var/lib/docker` + `/var/lib/docker/overlay2`, asserts the dictionary JOIN surfaces both, the prefix-bound GLOB rejects `/var/lib/dockerfoo`, and paths return as `bytes`.

**1. [Rule 3 - Blocking] Repointed the remaining old-blob-schema test fixtures onto the dictionary**
- **Found during:** Full-suite verification (the plan's definition of done is a fully-green suite; 70 prior failures spanned six test files, not only the two the plan names).
- **Issue:** `test_cli_report_commands.py`, `test_diagnostics_df_index.py`, `test_cli_collect.py`, `test_scanner_semantics.py` built `DirectoryAggregate(name=...)` (dropped in Plan 02) and/or read `SELECT path FROM directory_sizes` / inserted into the dropped blob columns.

- **Commit:** `83f82c1`.

**2. [Rule 3 - Blocking] Deleted the v1->current blob-schema migration idempotency test**
- **Found during:** Full-suite verification (`tests/test_grouping.py::test_initialize_database_migrates_v1_rows_idempotently...`).
- **Issue:** The test seeded a v1 blob-schema DB (`directory_sizes` with `name BLOB`) and expected `initialize_database` to migrate it; the new schema DDL references `path_id`, so `initialize_database` raises `no such column: path_id` over a v1 DB. This is an old-DB-migration scenario that no longer exists — Plan 02 ships a fresh schema with no down-migration, and the project's no-backward-compat policy forbids supporting prior-version state.
- **Fix:** Deleted the test and its now-dead `V1_SCHEMA_SQL` constant; replaced it with `test_initialize_database_is_idempotent_and_rolls_back_on_schema_failure`, which asserts the surviving real behavior (idempotent re-init on a current DB; rollback leaving `user_version` 0 and no `snapshot_mounts` on a schema explosion) without seeding a legacy DB.
- **Commit:** `83f82c1`.
