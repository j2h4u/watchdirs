# Phase 03.1 Plan 02: Flat Path-Dictionary Schema Spine Summary

Landed the structural compression change every later plan depends on: a flat `paths(id, path UNIQUE)` dictionary, `directory_sizes` rewritten to carry int `path_id`/`parent_id` FKs with int-keyed indexes, an in-process dedup cache (`_resolve_path_id`), the dropped `name` column, SCHEMA_VERSION 3, and virgin-connection `page_size`/`auto_vacuum`/`application_id` PRAGMAs — no migration shim, fresh schema only.

## What Was Built

- **`schema.sql`** — new `paths(id INTEGER PRIMARY KEY, path TEXT NOT NULL UNIQUE)`; `directory_sizes` drops `path`/`parent_path`/`name` BLOBs and gains `path_id INTEGER NOT NULL REFERENCES paths(id)` + nullable `parent_id INTEGER REFERENCES paths(id)`. Indexes swapped to `directory_sizes_pathid_snapshot_idx (path_id, snapshot_id)` and `directory_sizes_snapshot_parent_idx (snapshot_id, parent_id)`; `directory_sizes_snapshot_size_idx` unchanged. `snapshot_mounts` and its three indexes untouched.
- **`connection.py`** — `open_connection` sets `page_size=8192`, `auto_vacuum=FULL`, `application_id=WATCHDIRS_APPLICATION_ID (0x57645273)` on the virgin file (size==0) before any table, then the existing WAL/foreign_keys/busy_timeout. Module constants `WATCHDIRS_PAGE_SIZE`, `WATCHDIRS_APPLICATION_ID`.
- **`migrations.py`** — `SCHEMA_VERSION = 3`; new `_resolve_path_id(connection, cache, path) -> int` (cache hit → `SELECT id FROM paths WHERE path=?` → on true miss `INSERT INTO paths(path)` then `lastrowid`; binds raw `sqlite3.Binary` bytes; no `OR IGNORE`/`RETURNING`). `insert_directory_rows` builds a per-call `dict[bytes,int]` cache, resolves each row's path/parent_path to ints, and `executemany`s the int-keyed INSERT, preserving the `INSERT_BATCH_SIZE` batching. `initialize_database` left unchanged (no down-migration).
- **`models.py` / `scanner.py`** — `DirectoryAggregate.name` removed; `name=` dropped from `_directory_row`, `_skipped_directory_row`, and `aggregate_entry`; `_path_name_bytes` deleted.
- **Tests** — `tests/test_path_dictionary.py` (cache-hit-issues-no-SQL, already-seen-path-resolves-to-existing-id, non-UTF-8 byte-exact roundtrip); `tests/test_migrations_dictionary.py` (shared-path dedup, path_id/parent_id resolve back, `[10000, 5]` batch boundary via a recording connection); `tests/test_db_schema.py` rewritten to assert the dictionary schema, int indexes, SCHEMA_VERSION 3, and the virgin PRAGMAs.
