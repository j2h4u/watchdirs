<objective>
Build the path-churn / cardinality measurement harness (D-08 method C) that runs on the
CURRENT blob `directory_sizes` schema, before any rewrite. This is the FIRST deliverable
of the phase: the measured churn rate and dedup_ratio are the ROI-determining unknown
(panel-flagged in CONTEXT.md) and the numeric input to the D-09 per-snapshot byte budget
gate that Plan 04 finalizes.

Purpose: Set the size target against REAL data, not an idealized 0%-churn model. High
dedup_ratio / low churn ⇒ ~5–6× win; high churn ⇒ win collapses toward index int-ization.
Output: `src/watchdirs/bench/churn.py` (runnable via `uv run python -m watchdirs.bench...`)
and `tests/bench/test_churn.py` proving the SQL on a synthetic fixture.
</objective>
