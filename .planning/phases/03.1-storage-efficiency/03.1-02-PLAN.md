<objective>
Land the flat path-dictionary schema spine (D-01, D-02, D-03, D-04, D-05) with the
RESEARCH-corrected dedup pattern and PRAGMA ordering. This is the structural change every
other plan depends on: `paths(id, path)` dictionary, `directory_sizes` carrying int
`path_id`/`parent_id`, dropped `name`, int-keyed indexes, an in-process dedup cache, and
virgin-connection `page_size`/`auto_vacuum`/`application_id`.

Purpose: shrink watchdirs's own on-disk footprint — path bytes go from ~412× (per-snapshot)
to ~2× total (table + UNIQUE autoindex). Hard prerequisite for Phase 4 scheduling.
Output: rewritten `schema.sql`, `migrations.py` (`_resolve_path_id`, SCHEMA_VERSION 3),
`connection.py` PRAGMAs, `models.py`/`scanner.py` with `name` removed, and unit tests.

Engine: SQLite only — NO engine change (D-06). DuckDB/SurrealDB stay shelved; the dictionary
encoding done here in SQLite IS the compression win, on a 25-year-stable single-file forensic
format. (Escalation to DuckDB is considered only by Plan 04 if the D-09 gate misses by a wide
margin, per D-07 — not in this plan.)

⛔ NO BACKWARD COMPATIBILITY: bump SCHEMA_VERSION to 3, ship fresh schema. NO migration
shim, NO fallback for the old BLOB columns, NO "old DB still loads" path. Dev DBs
regenerate. A removed `name` column hitting old code SHOULD fail-fast.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem path bytes → `paths` INSERT/SELECT | Arbitrary filename bytes (possibly non-UTF-8, attacker-influenceable via crafted filenames) cross into SQL during collect. |
| collect → on-disk SQLite file | The dictionary becomes the forensic record of what was on disk; lossy/incorrect roundtrip corrupts evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-02-01 | Tampering | `_resolve_path_id` SELECT/INSERT into `paths` | mitigate | Path bytes bound as parameters (`?`), never interpolated — ASVS V5. Acceptance criterion asserts no string-formatting of path into SQL. |
| T-03.1-02-02 | Tampering / forensic integrity | non-UTF-8 path roundtrip through `paths.path` | mitigate | Bind raw `bytes` (not decoded `str`); `test_non_utf8` asserts byte-exact lossless roundtrip (Pitfall 2). Declaring TEXT but binding bytes is verified lossless. |
| T-03.1-02-03 | Tampering / forensic integrity | `application_id` magic number (D-05) | mitigate | `PRAGMA application_id` set to a fixed forensic file-type magic on virgin connection, supporting later file-type identification. |
| T-03.1-02-04 | Repudiation | removed `name` column hitting stale code | accept (fail-fast) | Per project rule, a stale `name` reference SHOULD raise. No compat shim. The grep acceptance criteria ensure no live reader remains. |
</threat_model>
