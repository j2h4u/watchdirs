# Phase 03.1 Plan 05: Collect Observability Summary

**collect emits INFO progress (dirs/rate/ETA from a monotonic rate) and one structured end-summary record (dirs/duration/db_bytes) to stderr, while stdout stays pure JSON — proven by a stderr-vs-stdout-purity test.**

## Performance

- **Duration:** ~20 min
- **Completed:** 2026-06-15
- **Tasks:** 2 (TDD: RED + GREEN)
- **Files modified:** 2 (1 created, 1 modified)

## Accomplishments
- `configure_collect_logging(verbose)` binds a `logging.StreamHandler(sys.stderr)` to a `watchdirs.collect` logger — NEVER stdout — protecting the agent-facing JSON contract.
- `compute_eta` / `log_progress` derive rate and ETA from `time.monotonic()`; `log_summary` emits one structured record (`dirs`, `duration_s`, `db_bytes`).
- `db_bytes` read post-commit via `PRAGMA page_count` × `PRAGMA page_size` (live WAL-mode size).
- A4 ETA seed: the previous COMPLETE snapshot's `directory_sizes` count for the same root (rate-only on first scan).
- `--verbose` flag on the `collect` subcommand gates INFO observability; stdout JSON purity is independent of the flag.

## Files Created/Modified
- `tests/test_collect_observability.py` - stdout-JSON-purity + stderr-summary/progress assertions (subprocess via `run_repo_local`), a deterministic `compute_eta` unit test (injected elapsed, no sleeps), and a handler-binds-stderr-only assertion.
- `src/watchdirs/cli.py` - observability helpers + module logger; `--verbose` flag; `run_collect` wired to configure logging, seed the ETA estimate, emit per-root progress, and emit one summary in `finally`.

## Decisions Made
- **Progress cadence:** one line per scanned root. The single-pass `os.scandir` walk gives no mid-walk yield point, so per-root is the natural bounded cadence (also satisfies T-03.1-05-03 — no per-row logging).
- **Testable ETA:** `compute_eta` takes `elapsed` as an argument rather than reading a clock internally, so the rate-based formula is pinned deterministically without sleeps.
- **Summary placement:** emitted in `run_collect`'s `finally` block so it fires on success, error, and interrupt; `db_bytes` is read before `connection.close()`.
