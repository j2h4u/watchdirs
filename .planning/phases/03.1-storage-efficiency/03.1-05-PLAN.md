<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| collect runtime → stdout (JSON contract) vs stderr (logs) | Agents/scripts parse stdout as JSON; any log leakage to stdout breaks the machine-readable contract. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.1-05-01 | Tampering / contract integrity | collect stdout JSON purity | mitigate | Logging handler bound to `sys.stderr` ONLY; acceptance criterion asserts no stdout handler and stdout still parses as JSON. Protects the agent-facing contract. |
| T-03.1-05-02 | Information disclosure | progress/summary log lines (paths, counts) | accept | Lands in the local systemd journal under the operator's own unit (Phase 4); single-user host, no network surface. Logs counts/timings, not file contents. Proportional to offline CLI. |
| T-03.1-05-03 | Denial of service | excessive progress logging on huge scans | mitigate | Progress emitted at a bounded cadence (every N dirs / N seconds — D-11), not per-row, so log volume stays proportional. |
</threat_model>
