## Deviations from Plan

### Auto-fixed Issues (Rule 1, during Task 2 GREEN)

**1. [Rule 1 - Bug] VACUUM left bytes in the WAL, breaking stat() reconciliation**
- **Found during:** Task 2 GREEN — `page_count*page_size=229376` but `stat()=8192`.
- **Issue:** In WAL mode `VACUUM` rewrites pages but does not checkpoint them into the main DB file,
  so `os.path.getsize()` under-reports. The harness's own reconciliation assertion (correctly) fired.
- **Fix:** Added `PRAGMA wal_checkpoint(TRUNCATE)` after `VACUUM` in `measure_db_size` so the page
  store and `stat()` reconcile. **File:** `src/watchdirs/bench/size.py`. **Commit:** `796bc1f`.

**2. [Rule 1 - Bug] Test asserted dbstat-sum == page_bytes exactly (auto_vacuum overhead)**
- **Found during:** Task 2 GREEN.
- **Issue:** `auto_vacuum=FULL` keeps a pointer-map page the dbstat aggregate view does not attribute
  to a named object, so `SUM(dbstat.pgsize)` is legitimately one page short of `page_count*page_size`.
  The exact-equality assertion would never hold for a correct measurement.
- **Fix:** Relaxed to `dbstat_sum <= page_bytes` and `page_bytes - dbstat_sum < page_size*4` (within
  fixed page overhead), still proving the autoindex is counted. **File:** `tests/bench/test_size.py`.
  **Commit:** `796bc1f`.

**3. [Rule 1 - Bug] Unrealistically short fixture paths inverted the directional win**
- **Found during:** Task 2 GREEN — at 14-byte paths × 3 snapshots the NEW DB was larger than OLD.
- **Issue:** The dictionary win comes from not re-storing path bytes per snapshot; with tiny paths and
  few snapshots the dictionary's fixed overhead (separate `paths` table + UNIQUE autoindex + extra
  `parent_id` index) dominates. The win is real but only visible at realistic path length / recurrence
  (confirmed: 1.03x at 3 snaps → 2.67x at 40 snaps with deep paths).
- **Fix:** Fixture now uses a representative deep path and 8 snapshots so the directional win is honest
  within the unit-suite latency budget. The full ~412-snapshot measurement remains the operator's run.
  **File:** `tests/bench/test_size.py`. **Commit:** `796bc1f`.

**4. [Rule 1 - Bug] Size benchmark reused fixed scratch DBs without wiping them (false 16x regression)**
- **Found during:** Task 4 operator gate (first real run).
- **Issue:** The size benchmark reused fixed scratch DBs (`.bench_size/bench_{old,new}.sqlite3`) without
  wiping them, so a prior/crashed run's rows were appended on the NEW side (`CREATE TABLE IF NOT EXISTS`)
  or errored on the OLD side — the first real run gave a FALSE "dictionary 16x bigger" until cleaned.
- **Fix:** Delete the scratch DB plus its `-wal`/`-shm` sidecars before each build; added regression test
  `test_rerun_in_same_workdir_is_idempotent`. Full suite 188 passed. **Commit:** `a676f98`.

## Honest Caveats (recorded for downstream readers)

1. **Measured churn was 0** (back-to-back scans; no time-spaced scan series exists yet). The gate was
   therefore closed on a churn SWEEP (bracketing 0-40%) rather than a single measured churn rate. This
   is sound because the win holds across all plausible churn, but a future time-spaced `collect` series
   (D-08 bolt-on C) would pin the exact operating point.
2. **The absolute per-snapshot byte budget (49,152 B) is for the `/opt` 421-dir REFERENCE tree.** The
   production root (`/`) has far more dirs, so its absolute per-snapshot figure scales up; the budget is
   best read normalized (≈117 B/dir/snapshot). The final production absolute number pins on a real
   prod-root scan.
3. **Temp benchmark DBs live in `/tmp` (tmpfs):** `/tmp/wd-opt.db`, `/tmp/wd-home.db` — not committed;
   regenerate via `collect` if needed.

  drop on /home and the posix_fadvise fallback on /opt, as designed.

  `page_count*page_size == stat()` reconciliation; autoindex counted (Pitfall 6) so the gate is set
  against real measured bytes. The idempotency bug (Deviation 4) was caught precisely because the
  integrity assertions surfaced the false inflated figure.
- **T-03.1-04-03** (tampering): path bytes bound as `?` parameters in both the OLD and NEW inline inserts;
  no interpolation.
- **T-03.1-04-04** (info disclosure, benchmark stdout): accepted — local dev tool, no network surface.
