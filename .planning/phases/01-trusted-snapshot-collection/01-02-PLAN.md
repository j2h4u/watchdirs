<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Configured roots -> scanner boundary | Validated paths become filesystem traversal inputs. |
| Collector -> SQLite | Snapshot evidence crosses from volatile filesystem reads into durable state. |
| SQLite schema -> later phase queries | Stored columns and indexes become the contract for diff/report phases. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-04 | Repudiation | `snapshots.status` and `snapshots.error` | High | mitigate | Persist `complete`/`partial`/`failed`, timestamps, notes, and fatal errors per D-11 through D-14. |
| T-01-05 | Tampering | SQLite migration lifecycle | Medium | mitigate | Use explicit `PRAGMA user_version` migrations and transaction-controlled snapshot finalization. |
| T-01-06 | Tampering | directory row contract | Medium | mitigate | Insert named `DirectoryAggregate` fields matching schema so byte/count semantics remain auditable. |
| T-01-07 | Denial of Service | per-row SQLite inserts | High | mitigate | Use `executemany()` batches of 10000 directory rows inside one transaction rather than one INSERT per aggregate. |
| T-01-08 | Repudiation | interrupted collection | High | mitigate | SIGINT/SIGTERM finalization records failed snapshots with `finished_at` and an interrupt error before exit. |
| T-01-09 | Tampering | non-UTF-8 path storage | High | mitigate | Store directory path identity as raw BLOB values; display decoding happens only at JSON/reporting boundaries. |
| T-01-SC | Tampering | package installs | Low | mitigate | No package-manager install task exists; stdlib runtime avoids package legitimacy risk in this plan. |
</threat_model>
