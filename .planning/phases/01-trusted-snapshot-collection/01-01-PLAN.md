<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| CLI args -> config loader | User-provided paths and notes enter the collector process. |
| Config file -> command dispatcher | TOML roots/excludes determine what later collection code may traverse. |
| Repo-local executable -> Python module | Local shell invocation crosses into package code without install bootstrap. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-01 | Tampering | `config.load_config` roots | High | mitigate | Validate absolute roots, reject empty and overlapping root sets, and require roots from config/CLI test fixtures per D-06 and D-20. |
| T-01-02 | Spoofing | command surface | Medium | mitigate | Test both `./watchdirs collect` and `PYTHONPATH=src python3 -m watchdirs collect` per D-26 so Phase 1 does not depend on an unverified installed command. |
| T-01-03 | Information Disclosure | JSON CLI errors | Low | accept | Output is local operator evidence; Phase 1 does not transmit data or expose a service. |
| T-01-04 | Elevation of Privilege | future timer config permissions | Medium | transfer | Phase 1 is user-run, but the 01-01 summary must call out that Phase 4 service work verifies config owner/mode before any root-run timer consumes TOML roots. |
| T-01-SC | Tampering | package installs | Low | mitigate | No package-manager install task exists; stdlib runtime avoids package legitimacy risk in this plan. |
</threat_model>
