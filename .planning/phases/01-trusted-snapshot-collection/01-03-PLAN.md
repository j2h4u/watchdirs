<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Filesystem entries -> scanner | Untrusted filenames, symlinks, hardlinks, and special files are inspected. |
| Scanner errors -> persisted evidence | OS errors must become durable evidence instead of disappearing. |
| Scanner aggregate rows -> SQLite | Computed byte counts and hierarchy become later forensic input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-05 | Tampering | symlink traversal | High | mitigate | Always stat with `follow_symlinks=False` and never recurse into symlinked directories. |
| T-01-06 | Tampering | hardlink accounting | Medium | mitigate | Deduplicate physical bytes by `(st_dev, st_ino)` per snapshot and test path-count semantics separately. |
| T-01-07 | Repudiation | path-level errors | High | mitigate | Persist scan errors and downgrade snapshot state to `partial` when useful data exists. |
| T-01-08 | Denial of Service | recursive traversal | High | mitigate | Use iterative post-order traversal and a 1500-depth regression test instead of recursive DFS. |
| T-01-09 | Denial of Service | hardlink dedup memory | High | mitigate | Default `hardlink_dedup_max_entries` to `500000`, enforce it for the exact inode set, and record a durable resource error when exceeded. |
| T-01-10 | Tampering | non-UTF-8 path identity | High | mitigate | Store path identity as raw bytes and decode only for JSON/error display. |
| T-01-11 | Tampering | configured exclude paths | Medium | mitigate | Honor `exclude_paths` in `ScannerOptions` and record skipped-path evidence. |
| T-01-SC | Tampering | package installs | Low | mitigate | No package-manager install task exists; scanner uses Python stdlib only. |
</threat_model>
