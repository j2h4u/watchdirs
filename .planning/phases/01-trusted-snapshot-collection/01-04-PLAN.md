<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| `/proc/self/mountinfo` -> mount parser | Kernel-provided mount rows enter policy code and may include escaped paths. |
| Mount classifier -> scanner descent | Policy decisions determine whether traversal enters a subtree. |
| Config include policy -> classifier | Operator-configured filesystem includes can override default skips. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-09 | Spoofing | mount parser | High | mitigate | Parse `/proc/self/mountinfo` fields directly, handle escaped paths, and use longest mountpoint matching. |
| T-01-10 | Tampering | mount classifier | High | mitigate | Default-skip pseudo, transient, overlay, and namespace filesystems named in D-09; require explicit config include to override. |
| T-01-11 | Denial of Service | scanner descent into virtual trees | High | mitigate | Check mount policy before descent and prune skipped mountpoints. |
| T-01-12 | Repudiation | skipped mount evidence | Medium | mitigate | Record skip reasons in scan errors or aggregate row errors for later reporting. |
| T-01-13 | Denial of Service | bind mount cycles | High | mitigate | Track visited mount IDs or `(major_minor, root, mount_point)` tuples and prune re-entry with recorded skip evidence. |
| T-01-14 | Tampering | cross-filesystem traversal | High | mitigate | Default `one_filesystem` mode prunes child `st_dev` changes so `/` scanning matches D-07 and `du -x`. |
| T-01-SC | Tampering | package installs | Low | mitigate | No package-manager install task exists; mount policy uses Python stdlib only. |
</threat_model>
