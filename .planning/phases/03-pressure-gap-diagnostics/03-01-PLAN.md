## Contract

`df-vs-index` success JSON must include `ok`, `command`, `snapshot_selector`, `limit`, `effective_limit`, `filesystems`, `summary`, and `warnings`. Each filesystem section includes storage-domain label, selected snapshot ids, snapshot finished-at min/max, maximum snapshot age seconds, `df_bytes.size`, `df_bytes.used`, `df_bytes.free_total`, `df_bytes.avail_unprivileged`, `indexed_visible_disk_bytes`, `indexed_visible_apparent_bytes`, `indexed_visible_path_count`, `indexed_root_paths`, `indexed_mount_points`, partial/unknown counters, `filesystem_scope_extends_beyond_indexed_roots`, `coverage_reason_codes`, `unattributed_bytes`, `unattributed_ratio`, `over_indexed_bytes`, `over_indexed_ratio`, `likely_reasons`, `verification_commands`, and `truncated`.

Indexed totals are storage-domain totals, not root totals. `query_indexed_storage_domain_totals()` must select one latest usable current snapshot per configured root when `snapshot_selector="latest"` and must not sum multiple current snapshots for the same root. For each selected snapshot, it must load persisted `snapshot_mounts`, resolve directory rows by longest persisted mount-prefix, and compute non-overlapping domain totals. The acceptable implementation is either:

1. identify storage-domain boundary rows where a row's resolved domain differs from its parent row's resolved domain, then subtract descendant boundary aggregates from ancestor aggregates before summing each domain; or
2. an equivalent SQL/Python aggregation that proves every indexed `disk_bytes` aggregate contributes to at most one storage-domain total.

This must handle roots that span nested submounts and roots that are themselves only subtrees of a larger filesystem.

`build_df_index_diagnostic()` must call `statvfs()` only for storage domains returned by `query_indexed_storage_domain_totals()`, using the selected domain mount point or another persisted path that belongs to that domain. It must not enumerate every live mount on the host in df-vs-index/report contexts.

When a storage-domain's live filesystem scope is broader than the indexed roots, set `filesystem_scope_extends_beyond_indexed_roots=true` and add a bounded reason such as `indexed_roots_are_subtrees_of_filesystem`. In that state, a material remainder is primarily a partial-coverage fact; it must not automatically emit `deleted_open_file_suspected`. Deleted-open suspicion is allowed only when indexed roots cover the storage-domain filesystem scope, or when independent deleted-open evidence from the dedicated command is supplied by a later summary step.

Material mismatch defaults: `MISMATCH_MIN_BYTES = 1 * 1024 ** 3` and `MISMATCH_MIN_RATIO = 0.05`. A mismatch is material when both the byte floor and ratio threshold are met. Ratios use filesystem used bytes as the denominator when positive.
