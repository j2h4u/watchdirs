## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI to Docker daemon | Docker CLI reads daemon state and may fail due to permissions or daemon availability. |
| Docker output to parser | Structured output may be line-delimited, blank, malformed, or version-dependent. |
| Docker evidence to recommendations | Reclaimable labels are evidence, not approval for cleanup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Tampering | Docker subprocess invocation | mitigate | Use fixed read-only argv and `shell=False`; never interpolate user strings into commands. |
| T-03-12 | Repudiation | Docker evidence completeness | mitigate | Emit source command, availability fields, exit warnings, and parser warnings. |
| T-03-13 | Denial of Service | Docker/build-cache output volume | mitigate | Enforce top-N limits and truncation metadata. |
| T-03-14 | Tampering | operational next checks | mitigate | Render verification commands only; no Docker mutation commands. |
| T-03-15 | Information Disclosure | local Docker paths/images | accept | Local forensic CLI intentionally surfaces local evidence to the operator/agent. |
| T-03-SC | Tampering | package installs | mitigate | No package-manager install task exists; use stdlib and installed Docker CLI when present. |
</threat_model>
