## JSON Contract Sketch: `diff`

- Success envelope: `ok`, `command="diff"`, `since`, `limit`, `effective_limit`, `group_by`, `pairs`, `rows`, `classification_counts`, `warnings`.
- `pairs` is an array of same-root pair metadata with `root_path`, `baseline` snapshot (`id`, `started_at`, `finished_at`, `status`, `error`), `current` snapshot, and warning codes.
- `rows` is one global frontier list after all valid pair candidates are merged, pruned, sorted, and limited. Each row includes `root_path`, `snapshot_pair` ids, `path`, `path_bytes_hex`, `depth`, `classification`, `previous_disk_bytes`, `current_disk_bytes`, `disk_bytes_delta`, `previous_apparent_bytes`, `current_apparent_bytes`, `apparent_bytes_delta`, `suppressed_descendant_count`, `suppressed_ancestor_count`, `reason`, and optional `group`.
- JSON error envelope uses `{"ok": false, "error": {"code": "...", "message": "...", ...}}` for `invalid_since`, `invalid_limit`, `limit_too_large`, and `no_snapshot_pairs`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| CLI `--since` / `--limit` / `--group-by` -> selectors | Untrusted local arguments determine query scope and output size. |
| Snapshot metadata -> report trust | Pair selection can mislead if roots or statuses are hidden. |
| Raw diff SQL -> frontier pruning | Full recursive aggregate rows are reduced to compact evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-10 | Tampering | diff SQL selectors | mitigate | Parameterize snapshot ids, limits, and path values; map group selectors through code constants only. |
| T-02-11 | Repudiation | partial/failed scan evidence | mitigate | Exclude failed snapshots from usable pairs, allow partial snapshots with explicit warnings, and include status/error fields in pair metadata. |
| T-02-12 | Spoofing | cross-root snapshot pairing | mitigate | Resolve baseline/current pairs by identical `root_path`; merge outputs only after valid same-root pairs exist. |
| T-02-13 | Tampering | non-UTF-8 paths | mitigate | Keep query paths as BLOB bytes and render with surrogate-safe decoding plus `path_bytes_hex`. |
| T-02-14 | Denial of Service | unbounded diff/frontier output | mitigate | Validate `--limit` as 1..1000 with default 20, prune default output, and apply the final global limit after multi-root candidate merge; keep full recursive dumps out of default `diff`. |
| T-02-SC | Tampering | package installs | mitigate | No new package installs are planned; use Python stdlib and existing pytest only. |
</threat_model>
