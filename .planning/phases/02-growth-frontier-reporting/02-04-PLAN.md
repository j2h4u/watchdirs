## JSON Contract Sketches

### `report`

- Success envelope: `ok`, `command="report"`, `since`, `limit`, `effective_limit`, `group_by`, `pairs`, `warnings`, `classification_summary`, `group_summary`, `frontier`, `deleted_preview`.
- `classification_summary` includes counts and disk/apparent delta sums for `created`, `deleted`, `grown`, `shrunk`, and `unchanged`.
- `group_summary` must not double-count recursive parent/child aggregates: aggregate over selected frontier rows or another explicitly one-row-per-group slice, not over every recursive raw diff row.

### `deleted`

- Success envelope: `ok`, `command="deleted"`, `since`, `limit`, `effective_limit`, `pairs`, `warnings`, `rows`.
- Each row includes `root_path`, `snapshot_pair`, `path`, `path_bytes_hex`, `classification="deleted"`, `previous_disk_bytes`, `previous_apparent_bytes`, `current_disk_bytes=0`, `current_apparent_bytes=0`, and negative delta fields. Deleted output is a strict baseline-only list sorted by previous disk bytes and limited; it is not frontier-pruned.

### `explain-path`

- Success envelope: `ok`, `command="explain-path"`, `since`, `limit`, `effective_limit`, `depth`, `group_by`, `pairs`, `target`, `children`, `unshown_or_direct_disk_bytes_delta`, `unshown_or_direct_apparent_bytes_delta`, `warnings`.
- Path input is converted with `os.fsencode()` after path normalization, matched to exactly one selected root, and requires an exact target row. It explains the target subtree; it does not fuzzy-match arbitrary prefixes.
- JSON error envelope uses `{"ok": false, "error": {"code": "...", "message": "...", ...}}` for `path_outside_roots`, `path_not_indexed`, `ambiguous_root`, `invalid_depth`, `invalid_limit`, `limit_too_large`, and inherited pair-selection errors.

## Trust Boundaries

| Boundary | Description |
|---|---|
| CLI path input -> SQLite filters | `explain-path PATH` selects stored BLOB rows for a subtree. |
| Raw diff rows -> summary renderer | Classification counts and group summaries can mislead if labels or warnings are dropped. |
| User limits/depth -> query breadth | Report commands can produce too many rows without caps. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-15 | Tampering | `explain-path` path filter | mitigate | Normalize the CLI path, convert it with `os.fsencode()`, require an exact indexed target under one selected root, bind bytes as parameters, and return structured outside-root/not-indexed errors. |
| T-02-16 | Repudiation | report/deleted/explain evidence | mitigate | Include baseline/current ids, timestamps, root paths, statuses, errors, and warning codes in every payload. |
| T-02-17 | Information Disclosure | local path rendering | accept | Local CLI intentionally reports configured local paths for the operator/agent; JSON is not sent over a network by this phase. |
| T-02-18 | Denial of Service | unbounded report/depth output | mitigate | Validate `--limit` as 1..1000 with default 20 and `--depth` as 0..20 with default 1; use compact defaults and keep full recursive output out of default commands. |
| T-02-19 | Spoofing | storage-domain grouping | mitigate | Use persisted snapshot_mounts from the selected snapshot pair for mount/domain labels, and use root-relative BLOB path segmentation for top-level-subtree labels; never recompute historical mount labels from current mountinfo. |
| T-02-SC | Tampering | package installs | mitigate | No new package installs are planned; use Python stdlib and existing pytest only. |
</threat_model>
