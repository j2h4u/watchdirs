## JSON Contract Sketch: `top`

- Success envelope: `ok`, `command="top"`, `snapshot_selector`, `limit`, `effective_limit`, `group_by`, `sections`, `warnings`.
- `--snapshot latest` returns one section per `root_path`: each section contains `snapshot` metadata (`id`, `root_path`, `started_at`, `finished_at`, `status`, `error`), `warnings`, and up to `effective_limit` rows for that root.
- Numeric `--snapshot ID` returns exactly one section for that snapshot id.
- Every row includes `snapshot_id`, `root_path`, `path`, `path_bytes_hex`, `depth`, `current_disk_bytes`, `current_apparent_bytes`, `file_count`, `dir_count`, `error`, and optional `group`.
- `group.kind="mount"` labels the selected snapshot mount point; `group.kind="storage-domain"` uses key `major_minor + root + filesystem_type + mount_source` and labels `filesystem_type`, `mount_source`, `major_minor`, `root`, and matched `mount_point`.
- If `--group-by mount` or `--group-by storage-domain` finds no persisted mount prefix for a row, that row uses `group=null` and the enclosing section includes warning code `unknown_mount`; the command must not synthesize a fake mount or storage-domain label.
- JSON error envelope uses `{"ok": false, "error": {"code": "...", "message": "...", ...}}` for `invalid_snapshot_id`, `snapshot_not_found`, `no_usable_snapshots`, `invalid_limit`, and `limit_too_large`.

## Trust Boundaries

| Boundary | Description |
|---|---|
| CLI selectors -> query helpers | `--snapshot`, `--limit`, and `--group-by` values become SQLite query parameters or selector branches. |
| SQLite BLOB paths -> renderer | Raw path identity becomes JSON/text output. |
| Stored snapshot status -> report trust | Partial or failed evidence must stay visible to the agent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-02-05 | Tampering | `top --group-by` and `--snapshot` | mitigate | Use argparse choices for `root`, `top-level-subtree`, `mount`, and `storage-domain` plus numeric/latest selector parsing; never concatenate raw selector values into SQL. |
| T-02-06 | Tampering | top query path values | mitigate | Keep BLOB paths in query/dataclass layers and decode only in `render.decode_path()` with surrogate-safe behavior plus `path_bytes_hex`. |
| T-02-07 | Repudiation | snapshot status evidence | mitigate | Include snapshot id, timestamps, status, error, and warning list in JSON and text output. |
| T-02-08 | Spoofing | filesystem/storage-domain grouping | mitigate | Use persisted `snapshot_mounts` longest-prefix matching from snapshot id, not live mountinfo. |
| T-02-09 | Denial of Service | unbounded top results | mitigate | Require and cap `--limit` through validated integer parsing with a compact default. |
| T-02-SC | Tampering | package installs | mitigate | No new package installs are planned; use Python stdlib and existing pytest only. |
</threat_model>
